## Рекомендации по написанию обзоров

### Название файла
- Используйте латиницу или транслитерацию
- Будьте описательными
- Примеры: `master-i-margarita-review.md`, `sapiens-analysis.md`

### Теги
Используйте теги для категоризации:
- По жанру: `фантастика`, `нон-фикшн`, `детектив`
- По теме: `философия`, `история`, `технологии`
- По настроению: `мотивация`, `грусть`, `юмор`
- По типу: `анализ`, `впечатления`, `рецензия`

### Структура
- **Вступление** - зацепите читателя
- **Основная часть** - ваш анализ
- **Заключение** - чёткие выводы

### Стиль
- Пишите честно и от души
- Аргументируйте свои мнения
- Используйте примеры из книги
- Уважайте мнения других

### Спойлеры
Если обзор содержит спойлеры, обязательно:
1. Укажите `spoilers: true` в frontmatter
2. Добавьте предупреждение в начале
3. Используйте блок `:::` для скрытия особо важных спойлеров

После создания обзора, добавьте ссылку на него в список на странице [Все обзоры](/reviews/).
